# 100篇定律——在你写完100篇之前，不要判断自己适不适合

[]

---

大多数想开始做内容的人卡在同一个地方：写完前几篇，阅读量是个位数，觉得自己"不适合做这个"，放弃了。

100篇定律的核心洞见是：你的前100篇内容的唯一功能，不是获取流量、不是建立品牌、不是积累粉丝——而是"让你学会写"。

## 为什么是100篇

100不是魔法数字。但它足够多——多到你能经历以下阶段：

**1-20篇**：克服"写不出来"的恐惧。每篇都需要你鼓起勇气点"发布"按钮。你写完会反复检查、修改、焦虑。

**21-50篇**：形成基本的写作节奏。你不再需要"酝酿一周"才能写一篇。写作变成了一件"可以按计划完成"的事。

**51-80篇**：你开始听到"声音"。不是文字的声音——是你自己的声音。你开始辨认出"这是我真正想说的"和"这只是我在套别人的框架"之间的区别。

**81-100篇**：你变得不那么在乎每一篇的反馈。你有了"作品集"——不是某个单一爆款，而是一个持续的、可追溯的思考轨迹。人们不是因为你"写了一篇好文章"而关注你——而是因为你"一直在写、一直在想、一直在分享"。

## 为什么要写完100篇才有资格判断

因为你前20篇的判断是噪音。你可能第3篇就爆了（运气），然后第4-20篇全部回归个位数，然后你觉得自己"水平退步了"（不是——是你的运气用完了）。你可能前50篇都默默无闻，第51篇突然爆了——不是因为第51篇写得特别好，是因为你前50篇积累的隐形能力在某一篇中集中爆发了。

在100篇之前，你不能区分"运气波动"和"真实能力"。100篇之后——你的样本量足够大，你可以开始做有意义的自我评估。

## AI让100篇变快了——但"写"本身没有变容易

AI可以帮你写得更快。但"写"的核心——决定写什么、怎么组织、什么是真正值得说的——仍然需要你的人类判断力。没有这个判断力，AI只是让你更快地产出平庸内容。

100篇定律的AI时代版本：用AI加速"写的速度"，但不要用AI替代"想的过程"。笔可以换，脑子不能外包。

## 伴读引导

**读前自问**：

1. 这篇文章的核心主张，在你自己身上能找到证据还是反例？
2. 如果你要向一个完全没背景的朋友解释这篇的核心观点，你的第一句话会是什么？
3. 文中引用的《流量的本质与暴力》——你读过其中任何一本吗？如果没读过，单从这篇文章的判断，你最想先翻哪一本？

**读后检验**：

- 如果你读完觉得"有道理"但没有具体的"我要试试"——那这篇文章可能只是帮你强化了已有认知，没有产生新的行动可能性。
- 如果你在某一段停下来想了超过10秒——把那一段标记出来。那是真正触动你的地方。
- 如果全文没有任何地方让你想停下来反驳——可能你读得太快了，或者文章太"顺"了。

**伴读笔记模板**：

| 段落 | 我的反应 | 联想 | 不同意 |
|------|---------|------|--------|
| | | | |

*伴读方法来自 ljg-read — 不替你读，陪你走进去。扫清语言障碍只是起手式，真正的活儿是让你撞见自己从没想过的问题。*


## 延伸：Dan Koe 个人资产飞轮

Dan Koe 将内容创作提炼为一个可操作的飞轮：

1. **写反愿景**：先写清楚你不想要什么——这比写你想要什么更容易，也更真实。
2. **选"过去的自己"为受众**：写给三个月前的自己。你不需要"用户研究"——你就是用户。
3. **输入压缩成问题地图**：把读过的书、学过的课压缩成你能问自己的问题。不是"我知道什么"而是"我能回答什么问题"。
4. **每周公开输出**：不是偶尔发一篇，是每周。复利需要连续的时间t。
5. **把高频问题产品化**：当10个人问了你同一个问题——这个问题值得变成一个产品。

这个飞轮和"100篇定律"完全契合：前99篇是飞轮的第1-4步，第100篇是第5步。
