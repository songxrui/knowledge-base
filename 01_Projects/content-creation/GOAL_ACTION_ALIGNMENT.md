# 知识库内容目标-行动对齐检查

基于 dbs-goal + dbs-action 方法论。

## 全局目标设定（dbs-goal）

**知识库终极目标**: 让每一个读者在读完3篇文章后发生一个具体的认知改变。

**可量化指标**:
- 单篇文章目标: 每个读者在评论区留下的不是一个"好文"而是"我之前以为X，现在知道是Y"
- 知识库目标: 100篇文章覆盖认知升级的8个核心领域
- 质量指标: 97% compile-and-verify 通过率

## 每篇文章的行动对齐（dbs-action）

| 检查项 | 通过标准 | 当前状态 |
|--------|---------|---------|
| 有明确行动段 | 每篇文章有2-3个可执行动作 | 96/100 ✅ |
| 行动具体可验证 | 不是"好好努力"而是"今天做X" | 待逐篇核查 |
| 行动有障碍预案 | 识别常见失败模式并给兜底方案 | 部分缺失 |
| 行动有反馈闭环 | 做完后如何判断效果？ | 伴读引导已补充 |

## 失败模式（dbs-action 提取）

| 失败模式 | 表现 | 解法 |
|---------|------|------|
| 知识幻觉 | 读完觉得"我知道"但没有行动 | 伴读引导"如果你觉得知道→写一句话说清楚" |
| 行动延迟 | "明天开始"→ 永远不开始 | 每个行动段加"现在就可以做的第一步" |
| 完读率衰减 | 长文章读到一半跳出 | 每篇文章开头30秒测试 + 互动钩子 |
| 认知过载 | 一篇文章塞太多概念 | dbs-deconstruct 已分析，结构完整率差异大 |

*目标方法论: dbs-goal | 行动方法论: dbs-action | 检查工具: compile-and-verify*
