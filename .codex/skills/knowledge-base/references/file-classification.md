# 文件分类推荐体系

> 第一性原则：分类服务于检索，不是展示。最好的分类是"打开即找到，不需要记"。

## 快速决策树

```
新内容进来
├─ 能直接变成一个可完成的项目/任务？
│  └─ YES → 01_Projects/<project-name>/
├─ 属于持续维护的领域？（健康/财务/认知/关系...）
│  └─ YES → 02_Areas/<domain>/
├─ 是外部参考资料/素材？（书籍/文章/视频/课程...）
│  └─ YES → 03_Resources/<type>/<source>/
├─ 是原子化思考/概念？（可独立链接复用）
│  └─ YES → zettel/
└─ 已完成/不再活跃/过时？
   └─ YES → 04_Archive/
```

## 五维分类体系

### 维度一：按领域（Domain）— 个人知识体系

适用场景：持续维护的个人知识领域，内容会不断增长和更新。

| 领域 | 路径 | 典型内容 | 判定标准 |
|------|------|---------|---------|
| 健康 | `02_Areas/health/` | 运动记录、饮食、体检、睡眠 | 涉及身体/心理健康管理 |
| 认知 | `02_Areas/cognition/` | 思维模型、学习方法、决策框架 | 涉及"如何思考/学习" |
| 财富 | `02_Areas/wealth/` | 投资笔记、财务规划、收入流 | 涉及金钱/资产/投资 |
| 生产力 | `02_Areas/productivity/` | 工具流、时间管理、SOP | 涉及"如何更高效做事" |
| 自媒体 | `02_Areas/self-media/` | 内容策略、选题库、平台分析 | 涉及内容创作/分发 |
| 心理学 | `02_Areas/psychology/` | 情绪管理、人际、自我认知 | 涉及心理机制/行为 |
| 关系 | `02_Areas/relationship/` | 家庭、朋友、社交网络 | 涉及人与人互动 |
| 历史社会 | `02_Areas/history-society/` | 历史解读、社会现象分析 | 涉及历史/社会/文化 |

**判定标准**：如果一篇内容可以被标记到一个明确的领域话题，且未来可能在该领域内继续积累 → 使用此维度。

---

### 维度二：按项目（Project）— 有明确目标和终点

适用场景：有明确目标、交付物和截止时间的临时性工作。

| 路径模板 | 示例 | 判定标准 |
|---------|------|---------|
| `01_Projects/<project-name>/` | `01_Projects/book-writing/` | 有明确可交付成果 |
| `01_Projects/<project-name>/research/` | `01_Projects/market-analysis/research/` | 项目的研究素材 |
| `01_Projects/<project-name>/drafts/` | `01_Projects/article-series/drafts/` | 项目的草稿阶段 |

**判定标准**：你能说出这个项目的"完成"是什么样子 → 使用此维度。项目完成后整体迁移到 `04_Archive/` 或提炼精华到 `zettel/`。

---

### 维度三：按媒介（Media Type）— 来源管理

适用场景：外部参考资料的来源分类，便于按来源类型检索。

| 媒介类型 | 路径 | 典型格式 | 元数据要求 |
|---------|------|---------|----------|
| 书籍 | `03_Resources/books/<书名>/` | 书摘、读后感、章节笔记 | 作者、出版年、ISBN |
| 文章 | `03_Resources/articles/` | 网页存档、标注、摘要 | URL、发布日期、作者 |
| 视频 | `03_Resources/videos/` | 转录、关键截图、时间戳笔记 | 平台、频道、时长 |
| 播客 | `03_Resources/podcasts/` | 转录、要点、时间戳 | 节目名、嘉宾、集数 |
| 课程 | `03_Resources/courses/` | 笔记、作业、证书 | 平台、讲师、完成日期 |
| 论文 | `03_Resources/papers/` | 摘要、方法、引用 | DOI、期刊、发表年 |
| 对话/会议 | `03_Resources/conversations/` | 记录、Action items | 日期、参与者、场合 |

**判定标准**：内容的核心价值来自外部来源，且你希望保留来源信息供日后核查 → 使用此维度。

---

### 维度四：按成熟度（Maturity）— 内容生命周期

适用场景：Zettelkasten 笔记系统的状态管理。

| 状态 | 含义 | 应满足条件 | 迁移规则 |
|------|------|----------|---------|
| `seedling` | 初步想法，待验证/扩展 | 至少1条来源+1个核心结论 | 30天未更新 → 归档或删除 |
| `budding` | 已有证据支持，正在生长 | ≥3条来源+双链≥2+已被引用 | 持续提炼后可升级 |
| `evergreen` | 成熟的原子知识单元 | ≥5条来源+双链≥5+经至少一次review | 不轻易改动，作为基础知识 |
| `published` | 已对外发布（公众号/飞书等） | 已发布URL+发布日期 | 发布后标记，原文保留 |

**判定标准**：笔记是"活的"且会持续迭代 → 使用此维度跟踪成长状态。

---

### 维度五：按受众（Audience）— 协作场景

适用场景：内容需要区分对内/对外，或面向不同协作者。

| 受众类型 | 标记方式 | 访问控制 |
|---------|---------|---------|
| 仅自己 | 默认，无需特殊标记 | 本地vault |
| 团队内部 | `tags: [internal]` | GitHub私有仓库 → 飞书内部Wiki |
| 公开发布 | `tags: [public]` | 经AI润色后 → 飞书门面/GitHub Pages |
| 特定客户 | `tags: [client:<name>]` | 按需分享 |

## 实际分类示例

### 示例1：微信读书《思考，快与慢》高亮

```
决策路径：
1. 是项目吗？→ NO
2. 是领域知识？→ YES (认知领域)
3. 是外部参考？→ YES (书籍)
4. 存在位置：03_Resources/books/thinking-fast-slow/ (原始书摘)
5. 提炼后的原子笔记：zettel/系统1和系统2的边界.md
```

### 示例2：一篇关于AI写作工具的测评文章

```
决策路径：
1. 是项目吗？→ 如果是系列测评的一部分 → 01_Projects/ai-tools-review/
2. 是领域知识？→ YES (自媒体/生产力)
3. 是外部参考？→ YES (文章)
4. 存在位置：03_Resources/articles/ai-writing-tools-2026.md
5. 提炼后的判断笔记：zettel/AI写作工具选型标准.md → 02_Areas/self-media/
```

### 示例3：个人年度财务回顾

```
决策路径：
1. 是项目吗？→ NO (每年都要做)
2. 是领域知识？→ YES (财富)
3. 存在位置：02_Areas/wealth/annual-review-2026.md
```

## 常见误区

| 误区 | 后果 | 正确做法 |
|------|------|---------|
| 按"心理学/编程/哲学"建顶层文件夹 | 笔记归属模糊，检索困难 | 用领域维度，归属到具体子话题 |
| 所有东西都放 zettel | zettel变成大杂烩 | zettel只放原子化概念，领域知识放Areas |
| 一个项目拆成10个文件夹 | 碎片化，找不到完整上下文 | 项目内最多3层：根/素材/产出 |
| 书籍笔记和书摘混放 | 检索时噪音大 | 书摘放03_Resources，思考提炼放zettel |
| 分类太细（>3层） | 维护成本超过收益 | 最多3层深度，再多用tag替代文件夹 |

## 工具建议

| 场景 | 推荐 | 原因 |
|------|------|------|
| 纯文件分类 | 文件夹+标签双轨 | 文件夹做粗分，标签做细分 |
| 跨维度检索 | `rg` / Obsidian search | 全文检索比完美分类更重要 |
| 定期整理 | Weekly Review | 7天周期，逐条决策归属 |
| 自动化 | weekly-review.ps1 | 自动检查Inbox堆积+孤岛笔记 |
