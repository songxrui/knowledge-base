# 知识库内容结构化系统 · 总纲

## 一句话定义

把 `D:\KnowledgeBase\01_Projects\content-creation\` 下159篇分散的MD文章，从"堆在文件夹里的库存"变成"可复用、可追溯、可重组、可持续生长的内容结构化工程"。

## 系统架构

```
_content-system/
├── 01-规则层/          # 字段规范、分类规则、关系规则、去重规则
├── 02-素材副本/         # 原始素材的只读副本（不修改原文）
├── 03-内容单元/         # 从素材中抽取的可复用内容原子
├── 04-主题地图/         # 跨域主题关联图 + 选题装配指南
├── 05-装配稿/          # 由内容单元组装的新稿件
├── 06-关系与去重/       # 内容关系索引、去重候选、冲突检测
└── 07-处理状态/         # 处理进度、覆盖范围、下一步
```

## 核心原则

1. **原始素材不改写** — 只复制副本处理
2. **默认目标不是"全部处理完"，而是"系统能用了"**
3. **结构先于规模** — 先验证结构，再批量推进
4. **每批处理后复盘** — 字段是否改动、关系是否改动、去重是否失控
