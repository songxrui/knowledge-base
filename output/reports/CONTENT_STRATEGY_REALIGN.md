## 一、平台矩阵（优先级排序）

### 🥇 主平台: 公众号 + 小红书图文

| 维面 | 公众号 | 小红书 |
|------|--------|--------|
| 内容形态 | 深度长文·3000-5000字 | 图文卡片·800-1200字 |
| 受众心智 | 信任型·订阅关系 | 探索型·算法推荐 |
| 内容策略 | 每周1篇旗舰文·系统化输出 | 每周2-3篇·碎片化钩子+引流公众号 |
| 变现路径 | 付费专栏/课程/社群 | 种草→私域→公众号 |
| 对标参考 | Dan Koe newsletter风格 | dontbesilent 图文密度 |

**公众号=信任资产沉淀池。小红书=流量入口+信任前哨。**

### 🥈 次平台: 抖音口播文稿

| 维面 | 策略 |
|------|------|
| 内容形态 | 60-90秒口播脚本·单点爆破 |
| 频率 | 每周1-2条 |
| 定位 | 从公众号长文拆解·一个观点=一条口播 |
| 对标 | Dan Koe YouTube shorts风格·短锐利·先给判断再拆框架 |
| 注意 | 口播文稿≠文章缩写。口播靠节奏+停顿+重音，文稿需标注【停顿】【强调】【语速】 |

**抖音=观点放大器。不追求粉丝量，追求"这人在公众号写了长文"的导流。**

### 🥉 随性平台: 推特/X

| 维面 | 策略 |
|------|------|
| 内容形态 | 短推文·长thread |
| 频率 | 不限·想到就发 |
| 定位 | 国内不能谈的话题·宣泄记录·不以盈利为目标 |
| 对标 | Naval风格·一句话洞见·不追求互动只追求记录 |
| 注意 | 敏感话题自行判断·AI不替你决定风险边界 |

## 二、对标框架融合: Dan Koe × dontbesilent

### Dan Koe 2026核心更新（exa验证·2026-01-03）

来源: letters.thedankoe.com "How I'd build a one-person business if I started over in 2026"

| 观点 | 对你的含义 |
|------|-----------|
| **信息产品已死→AI学习体验** | 不要只卖PDF课程。考虑"AI教练"型产品——把你的知识注入AI，让用户跟"你的AI分身"学习 |
| **Micro SaaS > 静态课程** | 长远方向：把你的方法论做成小工具。比如"六格复盘生成器""人生OS模板引擎" |
| **"你卖的是你大脑的第二版本"** | 你的Notion人生OS系统本身就是产品——把它从私用工具变成可分发产品 |
| **一人企业=营销+销售+产品一人全栈** | 你已有的技能栈（编程+写作+内容）恰好是Dan Koe说的一人企业全栈 |
| **深度通才>狭窄专家** | 你的跨领域（编程/交易/心理学/健康/关系）不是劣势——这是Dan Koe说的"deep generalism" |

### dontbesilent 内容哲学（dbs-content skill）

| 原则 | 应用 |
|------|------|
| **文字洁癖是底线** | AI辅助但绝不AI代写。每篇发布前humanizer-zh终审。 |
| **自媒体的本质是精神控制** | 标题和封面=认知劫持。不是标题党——是"让正确的人停下来" |
| **内容好坏=投入精力×正确理解** | 不要以量取胜。宁可一周1篇深度，不日更水文。 |
| **先有产品后有内容** | 内容的目标是"让读者想要你的产品"。先定义产品（人生OS模板/课程/社群），再围绕产品写内容。 |

### dbs 21-skill体系中你的高频skill

| skill | 用途 | 触发 |
|------|------|------|
| `dbs-content` | 内容全周期诊断 | 选题→结构→初稿→发布 |

| `dbs-deconstruct` | 拆解对标内容 | 学习dontbesilent/Dan Koe的文章结构 |

| `dbs-diagnosis` | 账号/内容策略诊断 | 月度内容审计 |
