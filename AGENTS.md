# KnowledgeBase — Agent 项目指令

> 最后更新: 2026-06-11 | 基于 dontbesilent/dbskill 工程实践 + 微软SkillOpt方法论

## 项目定位

这是董辉的个人知识库与内容生产工程。核心目标是：把本地内容资产（110+篇文章、35张深度卡、答案之书）搭建成可复用、可追溯、可重组的**内容工程**。

## 目录职责（Agent路由表）

| 目录 | 用途 | Agent行为 |
|------|------|----------|
| `01-内容生产/` | 选题管理+进行中草稿 | 进入=创作模式，先检索素材库再写 |
| `02-业务运营/` | 收入/产品/变现数据 | 进入=分析模式，输出数据洞察 |
| `03-内容素材库/` | 可复用的核心概念/金句/已验证稿件 | **写稿前必须先检索此目录** |
| `04-方法论沉淀/` | 从数据中提炼的平台方法论 | 每次发布后更新 |
| `05-数据统计/` | CSV数据表(每篇发布数据) | 发布后必须追加一行 |
| `06-原子库/` | JSONL知识原子(待建) | 按季度拆分 |
| `07-已发布/` | 发布后的内容归档 | 发布后从01-内容生产移动至此 |
| `media/` | 答案之书(book-v7)+DBS分析报告 | 只读参考 |
| `cards/` | 35张深度卡 | 只读参考 |
| `_meta/` | 工具脚本+审计报告 | 只读 |

## 核心工作流（发布一篇文章的完整流程）

1. **选题**：在 `01-内容生产/选题管理/00-选题记录.md` 记录想法
2. **检索素材**：AI先检索 `03-内容素材库/` 查找可复用内容
3. **写稿**：在 `01-内容生产/进行中/` 写草稿
4. **发布**：发布到公众号/小红书等平台
5. **记录数据**：在 `05-数据统计/数据统计表.csv` 追加一行
6. **归档**：将文稿移动到 `07-已发布/`
7. **反哺**：从数据中提炼洞察 → 更新 `04-方法论沉淀/`

## 禁止事项

- 不得直接修改 `media/flagship/` 中的答案之书（应创建新版本目录）
- 不得在根目录裸放文件（必须归入上述目录之一）
- 不得在未检索素材库的情况下从零创作
- 不得跳过数据记录步骤
