# Traffic Engineering — 底层流量工程

## 快速上手（3分钟）

1. 用户发来内容 → 先跑 Phase 1 注意力诊断（5个布尔判定）
2. 根据诊断结果选择 Phase 2/3/4 中不通过的项目执行优化
3. 输出结构化诊断报告 → 完成

---

## 核心哲学

**流量的本质，是人类注意力的时间切片。** (Stanley.team, 2026)

三个核心反转：
- 从"推出内容"反转为"让自己值得被注意" (Goldhaber, 1997)
- 从"寄生流量"反转为"自带流量"
- 从"标准化流水线"反转为"打破自己"

---

## 证据与反幻觉规则

执行本skill时，必须遵守以下约束：

1. **引用即标注**：所有平台数据/算法规则必须标注来源（Stanley.team 2026 / 学术论文 / 平台官方文档）
2. **推断即标记**：无法确认的判断必须以 `[推断]` 标注
3. **禁止编造数据**：不得伪造"X万人已收藏""Y%转化率"等social proof数字
4. **置信度三级**：
   - `[确认]`：有明确来源支撑
   - `[推断]`：基于已知框架的合理推演
   - `[经验]`：来自通用的内容运营经验，非特定数据
5. **不确定时明说**：如果无法判断某条建议是否适用，直接说"需要A/B测试验证"

**使用**：发布前检查、内容传播力诊断、跨平台适配、个人IP运营诊断、流量策略优化

## 执行流程

### Phase 1: 注意力第一性诊断

对内容执行5项布尔判定。任一不通过即进入对应优化Phase。

| # | 检查项 | 判定标准（布尔） | 不通过→ |
|---|--------|---------------|---------|
| 1 | 3秒钩子 | 前3行含反问/数据冲击/认知冲突之一？ | Phase 2 |
| 2 | 情绪锚点 | 含敬畏/愤怒/焦虑/幽默至少一种？ | Phase 2 |
| 3 | 一句话核心 | 删除所有修饰后，≤25字能说清核心？ | Phase 3 |
| 4 | 视觉首因 | 标题含数字/对比/悬念至少一种？ | Phase 4 |
| 5 | 能耗健康 | 全文核心概念≤3个？ | 拆分内容 |

> 判定依据：2023-2025短视频注意力研究 [确认]

### Phase 2: 心理学触发器优化

从以下5个传播力最强的触发器中选3-5个应用。完整21机制见 `references/psychology-triggers.md`。

**优先级**：好奇心缺口 > 损失厌恶 > 社会认同 > 框架效应 > 稀缺效应

| 触发器 | 内容公式 | 示例 |
|--------|---------|------|
| 好奇心缺口 | 标题埋悬念，中段开新坑 | "大多数人在第3步就放弃了——他们不知道的是..." |
| 损失厌恶 | "如果不做X，损失Y" | "每天少做这一件事，正在损失你的复利" |
| 社会认同 | 数字化证明 | 引用可验证的真实数据，禁止编造 |
| 框架效应 | 同一事实换表述 | "成功率90%" vs "失败率10%" |
| 稀缺效应 | 限时/限量/独家 | 仅用于真实限制，不得虚假制造紧迫 |

**情绪能量**：敬畏>愤怒>焦虑>幽默（高传播）/ 悲伤/平静/满足（低传播）

**检测命令**：`rg "好奇心|损失厌恶|社会认同|框架|稀缺" <file>`

### Phase 3: 分享动机强化

5大分享动机（Berger & Milkman, 2012）：

| 动机 | 检验问题 | 触发设计 |
|------|---------|---------|
| 社会货币 | 转发者会在朋友圈显得更聪明吗？ | 让内容成为"社交勋章" |
| 实用价值 | 内容对转发者的朋友有用吗？ | 清单/模板/步骤 |
| 情绪价值 | 引发了需要"倾倒"的强烈情绪吗？ | 敬畏/愤怒是传播燃料 |
| 触发器 | 绑定了高频日常场景吗？ | "每次开会前...""每晚睡前..." |
| 公开可见 | 分享行为本身是社交信号吗？ | 让分享有意义 |

**核心公式**：`转发概率 = 社会货币 × 实用价值 × 情绪唤醒` [确认]

同时命中"社会货币"+"实用价值" → 转发率5-8倍提升 [确认]

### Phase 4: 平台适配

根据目标平台选取对应规则执行。完整规则见 `references/platform-rules.md`。

**平台口诀**：
- **小红书**：封面即标题，搜索即流量，真实即信任
- **抖音**：前3秒定生死，情绪化定传播，垂直人设定留存
- **公众号**：搜索权重定冷启，收藏价值定长尾，深度信任定转化
- **X**：回复即流量，Thread即权重，中文即蓝海

**跨平台原则**：不是"一个内容发所有平台"，而是"同一世界观，不同形态"。深度长文→Threads(X)→短视频(抖音)→图文(小红书)。复制粘贴=偷懒，适配改造=专业。 [确认]

### Phase 5: 漏斗定位

判定内容阶段，确保标题/内容/CTA一致：

| 阶段 | 目标 | 内容策略 | 指标 |
|------|------|---------|------|
| TOFU 认知 | 曝光 | 宽泛有趣高分享 | CPM |
| MOFU 信任 | 建信 | 深度干货案例 | 互动率 |
| BOFU 转化 | 行动 | 限时优惠FAQ | CPA |

> ⚠️ MOFU内容用TOFU标题 = 点击高但信任流失 [经验]

### Phase 6: 终审

1. 陌生人在信息流中会停下吗？
2. AI能在10秒内生成类似内容吗？
3. 寄生流量还是建立资产？
4. 读者会主动分享吗？
5. 三个月后还有价值吗？

## 传播力评分卡（10维量化）

对内容执行10维评分，每维0-10分。总分决定传播潜力等级。

| # | 维度 | 0分 | 10分 | 得分 |
|---|------|-----|------|------|
| S1 | 3秒钩子力 | 平铺直叙开头 | 第1句就让人无法跳过 | /10 |
| S2 | 情绪唤醒度 | 无情绪 | 敬畏/愤怒≥1处可测量峰值 | /10 |
| S3 | 社交货币值 | 转发不会加分 | 转发后在朋友圈明显"加分" | /10 |
| S4 | 实用价值度 | 纯娱乐/纯观点 | 有可保存的清单/模板/步骤 | /10 |
| S5 | 平台匹配度 | 内容形态错配平台 | 形态×平台最优匹配 | /10 |
| S6 | 认知冲突度 | 用户"这个我知道" | 用户"等等，这不对吧？" | /10 |
| S7 | 金句密度 | 0句可截图转发 | ≥3句/千字可截图 | /10 |
| S8 | 漏斗一致性 | 标题/内容/CTA冲突 | 标题→内容→CTA完全一致 | /10 |
| S9 | 信息增量度 | AI可10秒生成同质内容 | 有不可替代的个人经历/洞见 | /10 |
| S10 | 时间抗衰度 | 3天后无价值 | 3个月后仍有阅读价值 | /10 |

**评级**：S(90+) 必爆 / A(75-89) 高概率传播 / B(60-74) 需优化 / C(<60) 建议重做

**使用**：每篇内容发布前打分。S级直接发，A级微调，B级回Phase 2-4优化，C级重做。

## 失败模式

| 场景 | 解法 |
|------|------|
| 内容好但没传播 | Phase 2+3：补情绪锚点和分享动机 |
| 标题党信任流失 | Phase 5：漏斗重定位，标题降级 |
| 跨平台复制粘贴 | Phase 4：逐平台适配改造 |
| 过早标准化 | Phase 1：注意力优先于流程 |
