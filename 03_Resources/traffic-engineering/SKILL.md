# Traffic Engineering — 底层流量工程

## 核心哲学

**流量的本质，是人类注意力的时间切片。** (Stanley.team, 2026)

你不是在"找流量"，你是在和全人类的内容创作者竞争同一批人的同一个24小时。
流量不是找来的，是赢来的。

### 三个核心反转

1. **从"如何把内容推出去"反转为"如何让自己值得被注意"** — 注意力经济(Goldhaber, 1997)
2. **从"寄生流量"反转为"自带流量"** — 投广告的钱花在提升自己；引流技巧的时间花在建立真实人脉
3. **从"标准化流水线"反转为"打破自己"** — SOP是你的经验结晶，也是思维天花板

---

## 何时使用

- 用户准备发布内容到小红书/抖音/公众号/X
- 用户问"为什么我的内容没人看/没有传播"
- 用户需要优化内容传播力
- 用户要做跨平台内容分发
- 用户需要做个人IP/媒体运营诊断

## 何时不使用

- 纯SEO/技术性搜索优化 → 使用 `seo` skill
- 内容写作本身 → 使用 `article-writing` / `content-engine`
- 视觉/设计问题 → 使用 `baoyu-cover-image` / `baoyu-infographic`
- 广告投放/付费流量 → 需额外商业分析

## 执行流程

### Phase 1: 注意力第一性诊断（5秒测试）

对用户的内容执行5项核心检查：

| # | 检查项 | 判定标准 | 不通过时的动作 |
|---|--------|---------|-------------|
| 1 | 3秒钩子 | 前3秒/前3行有认知冲突或信息缺口？ | 参考 `references/psychology-triggers.md` #好奇心缺口 |
| 2 | 情绪锚点 | 触发敬畏/愤怒/焦虑/幽默之一？ | 参考 `references/psychology-triggers.md` 情绪触发表 |
| 3 | 一句话核心 | 陌生人能一句话说清？ | 用5W模型检验：`references/communication-models.md` |
| 4 | 视觉第一印象 | 封面/标题在信息流中能让人停下？ | 检查平台规则：`references/platform-rules.md` |
| 5 | 能耗判断 | 用户理解这篇需要多少注意力？ | 简化表达、加对比、用故事 |

**输出格式**：
```
注意力诊断结果：
  ✅ 3秒钩子: {通过/不通过} — {具体原因}
  ✅ 情绪锚点: {情绪类型} — {触发位置}
  ⚠️ 一句话核心: 不通过 — 建议：{一句话版本}
  ...
```

### Phase 2: 心理学触发器优化

从21个机制中选取最适合内容的3-5个，逐一检查应用情况。
完整列表：`references/psychology-triggers.md`

**优先级排序**（按传播力）：
1. 好奇心缺口 > 2. 损失厌恶 > 3. 社会认同 > 4. 框架效应 > 5. 稀缺效应

**输出格式**：
```
心理学优化建议：
  已触发(3/5): 好奇心缺口(标题) + 社会认同(数据引用) + 权威偏误(来源标注)
  缺失: 损失厌恶 — 建议在{位置}添加"如果不做X会损失Y"
  缺失: 峰终定律 — 结尾不够强，建议加{具体强化方式}
```

### Phase 3: 分享动机强化

检查内容是否触发5大分享动机。详见 `references/communication-models.md`

**关键公式**：
```
转发概率 = 社会货币 × 实用价值 × 情绪唤醒
```
顶级内容同时命中"社会货币"和"实用价值"——转发率可以是单维的5-8倍。

### Phase 4: 平台适配

根据目标平台执行特定优化。详见 `references/platform-rules.md`

**平台一句话口诀**：
- **小红书**：封面即标题，搜索即流量，真实即信任
- **抖音**：前3秒定生死，情绪化定传播，垂直人设定留存
- **公众号**：搜索权重定冷启，收藏价值定长尾，深度信任定转化
- **X/Twitter**：回复即流量，Thread即权重，中文即蓝海

### Phase 5: 漏斗定位

判定内容属于TOFU/MOFU/BOFU哪个阶段，确保标题/内容/CTA与阶段一致。

| 阶段 | 目标 | 内容策略 | 核心指标 |
|------|------|---------|---------|
| TOFU 认知 | 曝光 | 宽泛、有趣、高分享性 | CPM、品牌搜索量 |
| MOFU 信任 | 建立信任 | 深度干货、案例对比、问题解答 | 互动率、私信获取量 |
| BOFU 转化 | 行动 | 限时优惠、社会证明、FAQ | CPA、ROAS |

> ⚠️ 常见错误：MOFU内容用TOFU标题 — 点击高但信任流失。

### Phase 6: 终审自检

发布前最后5问：

1. 陌生人在信息流中会停下吗？
2. AI能否在10秒内生成类似内容？
3. 是在寄生别人的流量还是建立自己的资产？
4. 读者会主动分享吗？
5. 三个月后还有阅读价值吗？

## 验证闭环

每条优化建议必须包含：
- **问题位置**：具体指出内容中的哪一段/哪一句
- **优化方向**：具体怎么改（不是"改得更有吸引力"而是"把第2段改成反问句开头"）
- **预期效果**：改了之后预期触发哪个心理学机制

**验证命令**：
```powershell
# 检查内容是否触发了心理学触发器
rg "损失厌恶|好奇心缺口|社会认同|框架效应|稀缺效应" <content-file>

# 检查平台适配
rg "完播率|封面驱动|搜索流量|收藏价值" <content-file>
```

## 失败模式

| 场景 | 表现 | 解法 |
|------|------|------|
| 内容质量好但没传播 | 缺少情绪锚点或分享动机 | 重点检查 Phase 2 + 3 |
| 标题党导致信任流失 | MOFU内容用了TOFU标题 | Phase 5 漏斗重定位 |
| 跨平台复制粘贴 | 同一内容发所有平台 | Phase 4 平台适配改造 |
| 追求标准化流程 | 粉丝<1000时过早标准化 | Phase 1 注意力优先 |
