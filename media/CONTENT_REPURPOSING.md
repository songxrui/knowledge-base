## 形态1: 自查清单 (Checklist)

### Skill健康度自检 (每项Yes/No, No≥3 → 需修复)

**可发现性**:
- [ ] SKILL.md的description包含≥4个具体触发词
- [ ] description包含"不适用场景"说明
- [ ] description包含≥2个正例 + ≥2个反例

**可追溯性**:
- [ ] 有.git目录且≥1个commit
- [ ] 有CHANGELOG.md记录版本历史
- [ ] 文件名/commit message不含虚假工时声明

**可验证性**:
- [ ] 有tests/目录且≥2个测试用例
- [ ] 至少1个正向测试(skill应该处理的)
- [ ] 至少1个边界测试(skill不应该处理的)

**可执行性**:
- [ ] SKILL.md ≤2000字 (Focused > Comprehensive)
- [ ] 包含≥1个可执行脚本/代码块
- [ ] 包含失败模式+兜底策略

**总计**: __/12 (≥9 = GA级, 6-8 = Beta级, <6 = Draft级)
