# 知识漫画 — Skill生态审计的概念漫画

> 产出方法: baoyu-comic (知识漫画) → 漫画分镜规格  
> 主题: "一个Draft Skill的自白"

---

## 分镜: 4格漫画

### 格1: "我出生了"
- 画面: 一个人(开发者)在键盘前, 屏幕上显示SKILL.md文件
- 对话框(开发者): "好了, 写完了。这个skill肯定好用。"
- 旁白: "一个Draft级skill诞生了。只有SKILL.md, 没有触发词, 没有git, 没有tests。"

### 格2: "我在等待"
- 画面: 一个AI Agent在工作, 旁边有一排skill盒子, 但这个skill的盒子是灰色的(隐身)
- 对话框(Agent): "需要优化文章... 我看看有哪些skill... (扫过灰色盒子) 没有匹配的。"
- 旁白: "74%的skill因为缺少触发词, 在Agent面前是隐身的。"

### 格3: "我终于被调用了! 但是..."
- 画面: Agent终于加载了这个skill, 但输出结果很差
- 对话框(Agent): "这个skill的输出... 还不如不用。"
- 数据标注: "OpenSkillEval: 最差skill拖累0.28分。选错skill < 不用skill。"

### 格4: "重生"
- 画面: 开发者回到键盘前, 屏幕上显示5步流程
- 对话框(开发者): "补触发词(5min) → git init(5min) → 精简模块(10min) → 加测试(5min) → CHANGELOG(5min)"
- 旁白: "30分钟后, Draft→GA。Skill完成了从草稿到产品的蜕变。"

---

## 漫画设计要点

- 风格: 极简线条, 类似xkcd风格
- 配色: 黑白为主, 关键数据用蓝色(#00d4ff)高亮
- 文字: 中文, 关键数字放大
- 用途: 社交媒体传播, 公众号配图
