## 规则层

| 规则 | 类型 |
|------|------|
| P0优先级: 能直接提升可发布率的操作 | 硬约束 |
| P1优先级: 能提升内容深度的操作 | 建议 |
| P2优先级: 锦上添花的操作 | 可选 |
| 每轮操作≤5项，防分散 | 效率规则 |
| 所有操作必须git commit留痕 | 审计规则 |

---

## 放弃条件
- 如果某个修改导致dbs-content评分下降 → 回滚该修改
- 如果某skill调用失败3次以上 → 记录失败,跳过
- 如果总操作时间超过2h → 暂停并汇报进度

## 验证方法
- 修改后dbs-content重新评分 ≥ 修改前
- content-auditor 5关全绿
- git log有完整commit序列
