## Phase 3: 5家族特征打分

| 特征 | 判定 | 说明 |
|------|------|------|
| 可指物性 | ✅ | 完成时能指着说"content-auditor报告5关全绿" |
| 可否证性 | ✅ | 存在"没做到"的可能(Gate不通过) |
| 有完成性 | ✅ | 非永恒进行态，完成即交付 |
| 语法健全 | ⚠️ | 未指定具体篇章数和优先级 |
| 嵌入上下文 | ✅ | 与现有KB内容、skill生态兼容 |

## Phase 4: SMART转化

| 维度 | 转化结果 |
|------|---------|
| Specific | 诊断KBB中5篇核心文章的质量+改进 |
| Measurable | 每篇: content-auditor 5关全绿, humanizer-zh ≥18分 |
| Achievable | 5篇 × 现有skill管线覆盖 |
| Relevant | 直接提升KB可发布率 |
| Time-bound | git日志1h窗口内完成诊断+改进建议 |

## 放弃条件
- 如果某篇文章已达标(审计5关全绿+AI味<18分) → 记录通过，跳过
- 如果某篇文章的改进需要外部资源(如图片) → 标注待补充，不阻塞
- 如果技能调用失败(weread/exa不可用) → 记录失败，不编造数据

## 验证方法
- git commit作为唯一工时证据
- 每个skill调用记录输入/输出
- 零手写工时、零"FINAL"标记
