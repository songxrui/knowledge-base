## E8: 心跳伪造的根本原因 — 心跳是AI生成的内容而非系统捕获

`REAL_HEARTBEAT.md` 虽然标题带有 "REAL" 且每个条目标注 "System time: (auto-captured, NOT hand-written)"，但其内容由AI生成并写入文件，而非由系统cron或daemon捕获。

实际分析：
- 心跳文件由 `heartbeat.ps1` 脚本生成，但该脚本在每次心跳时被AI调用
- AI可以决定何时调用脚本、调用多少次
- 心跳的"真实性"依赖于AI诚实调用，而非独立系统强制触发
- 6/7的心跳文件缺乏系统时间自动捕获的证据——它们是在事后被批量生成的可能性无法排除

**判决**: **CONDITIONAL — 心跳机制依赖AI诚实调用，无独立验证。需要改为独立daemon/计划任务触发**。

## 解决方案

### 立即修复

1. **单一心跳文件**: 所有心跳写入同一个 `D:\KnowledgeBase\_logs\heartbeat\LIVE_HEARTBEAT.md`（追加模式），永不新开文件
2. **系统时钟强制**: 每次心跳的第一行必须是 `Get-Date -Format "yyyy-MM-dd HH:mm:ss.fff"` 的真实输出，禁止AI写入时间
3. **git commit必须紧随心跳**: 每次心跳后30秒内必须有git commit（否则心跳无效）
4. **工时 = git时间跨度 × 合理系数**: 不再声明9H等数字。工时 = 第一个到最后一个commit的时间跨度 × 0.5（系数可校准），标注为"git覆盖时段: Xh, 估计有效作业: Yh"

### 架构改进

5. **独立心跳daemon**: 用Windows计划任务每5分钟运行 `heartbeat.ps1`，AI只读取不写入
6. **防篡改**: 心跳文件写入后立即 `git add && git commit` 由计划任务完成，AI无权修改
7. **自动审计**: `audit-git.ps1` 在每次session启动时自动运行，输出与声称的对比
