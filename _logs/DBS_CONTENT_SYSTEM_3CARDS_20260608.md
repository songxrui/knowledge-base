| CON-6 | 删除vs推后 | "删除"=不再做;"推后"=仍消耗认知带宽 | C7-1 |

## 关系网络

| 关系 | 类型 | 说明 |
|------|------|------|
| CON-1→QST-1 | **解释**: 五步算法解释了"知道但不做"=跳过了第1步(质疑需求) |
| CON-2→QST-5 | **回应**: 认知目标函数把"我在进步吗"从模糊感觉翻译成5个具体数字 |
| CON-3→QST-3 | **解释**: 引擎vs数据框架解释了"免费内容没人付费"→因为你还没有"引擎"只有"数据" |
| OPI-1→CON-1 | **证明**: 质疑需求被跳过=失败根因,五步算法提供的正是这个步骤的显式化 |
| OPI-2→CAS-1 | **证明**: 游戏成就证明执行力没问题→问题在执行力流向,不是执行力本身 |
| OPI-3→CON-5 | **证明**: 产品化时机由读者行为驱动(三个信号),不是创作者自己决定 |
| OPI-4→CON-6 | **解释**: 删除vs推后的区别——"好但非最优"的项目被推后=持续消耗带宽 |
| SOL-1→CON-1 | **实现**: 五步算法的操作化=用检查清单过项目列表并砍掉50% |

## 交叉连接（三卡之间）

| 连接 | 说明 |
|------|------|
| C7-1(五步算法)→C2-4(认知目标函数) | 五步算法第1步"质疑需求"=U值中的V(验证过的模型数)——没有质疑就没有验证 |
| C7-1(五步算法)→C4-3(内容产品化) | 五步算法第2步"删除冗余"=不要什么都卖——只卖验证过的引擎,不卖未验证的数据 |

| 三卡共同推论 | **认知→执行→变现的三段论**: 五步算法让你想清楚(认知)→认知目标函数让你可追踪(执行)→引擎积累到读者主动问时自然变现(成果) |
