## 提取结果

### 1. 节奏与句长

| 特征 | 数据 | 例子 |
|------|------|------|
| 短句密度 | 高。平均句长15-25字 | "这不是优化问题，是信任问题。"(8字) |
| 长句控制 | 偶尔出现30-40字长句，但绝不超过50字 | "当你在两个不同时间的两个不同会话里调用同一个skill，得到完全不同的结果时" |
| 段落长度 | 1-3句为主，极少超过5句 | - |
| 段落间节奏 | 短段→短段→中段循环，有明显的"呼吸节奏" | 每段独立一个观点 |

### 2. 压缩 vs 展开

| 倾向 | 证据 |
|------|------|
| 高度压缩 | "这不是优化问题，是信任问题" — 6个字重新定义了问题性质 |
| 数据即论据 | 每个论点后紧跟数字："74%的skill在Agent面前是隐身的" |
| 不展开废话 | 没有"随着XX的发展""在这个时代"等铺垫 |
| 结论敢下判断 | "草稿不能叫产品" — 直接判死刑，不留余地 |

### 3. 论证风格

| 模式 | 出现频率 | 典型句式 |
|------|---------|---------|
| 数字冲击 | 每3-4段1次 | "一共846个skill实例" |
| 反常识翻转 | 每篇2-3次 | "不是质量不好，是Agent根本不知道它们存在" |
| 论文引用 | 每篇2-3次 | "SkillRouter论文(arXiv 2603.22455v4)" |
| 对比论证 | 每篇1-2次 | ".codex 3.3% vs .agents 100%" |
| 行动号召 | 每篇1次(结尾) | "今天就可以做的一件事" |

### 4. 用词习惯

| 类型 | 常用 | 禁用 |
|------|------|------|
| 动词 | 避免抽象动词，用"扫了""审计了""发现" | 赋能/抓手/闭环/底层逻辑 |
| 连接词 | 极少用"首先其次最后"，用"第一第二第三"(有实质差异) | 综上所述/值得注意的是 |
| 语态 | 主动语态，"我审计了"而非"被审计" | - |
| 形容词 | 极度克制，每段≤1个形容词 | 深度/系统性/全方位 |

### 5. 情感基调

| 维度 | 风格 |
|------|------|
| 立场 | 明确，敢下判断。"不是""就是""不配"高频出现 |
| 姿态 | 审计者/揭穿者，"我审计了846个"建立专业权威 |
| 温度 | 冷静但不冷漠。有"残酷""致命"等情感词，但不煽情 |
| 自省 | 适度。"这不是苛刻"自我辩解，"我们刚走到这里"留白 |

### 6. 绝不做的事(负面特征)

| 绝不 | 证据(全量扫描确认为零) |
|------|------------------------|
| 绝不用"首先…其次…最后"三段式 | 全量0命中 |
| 绝不开头铺垫 | 直接扔数字/结论 |
| 绝不泛泛而谈 | 每个断言都有数字或论文引用 |
| 绝不开放式结尾 | 结尾必有三件事或行动号召 |
| 绝不用"赋能""抓手""闭环"等商业黑话 | 全量0命中 |
| 绝不自我感动式总结 | 无"综上所述""总而言之" |

## 声音画像一句话

**"审计者风格：用数字拆穿幻觉，用对比制造张力，用三件事收官。不铺垫、不煽情、不留开放结尾。"**
