## 形态1: 检查清单 (Checklist)

### Skill健康检查清单 (复制即用)

```
□ 我的核心skill(最常用的10个)description有触发词吗？
  检查方法: 读一遍description, 5秒内能说清它是干什么的吗？
  不能→花5分钟补4-8个触发词

□ 我的skill有版本控制吗？
  检查方法: ls .codex/skills/*/.git
  <.git存在→有; 不存在→git init && git add -A && git commit -m "init"

□ 我的内容管线能跑通吗？
  检查方法: 用1个选题走完整管线(weread→structure→write→humanize→verify)
  5阶段全绿→能; 任一环节失败→定位修复

□ 我的最新文章过内容审计了吗？
  检查方法: content-auditor 5关
  全绿→可发布; 有⚠️→修复后再发

□ 我最近30天新装的skill都评估过吗？
  检查方法: 新skill→跑一次→记录是否真的用了
  没用过→考虑移到.archived/
```
