# 逻辑哥交易系统

## 【材料识别】

- 主类型：理论框架型（市场结构认知框架）
- 次类型：方法论型（交易策略/资金管理/复盘训练的流程骨架）
- 夹杂：观点评论型（“xx%有效/99%胜率/牛市定义”等断言但缺验证口径）
- 判断依据：
    - 1–5课主要在定义“市场分层—制度—参与者—风险形态”的底层地图（变量与约束），典型框架。
    - 6课以后开始出现“筛选→触发→持仓→退出→复盘”的可执行流程组件，属于方法论/策略拼装。
    - 多处出现高确定性话术但缺样本/区间/回测口径，属于观点断言而非可验证事实。

## A｜理论框架层（你在什么场地交易、风险以什么形态出现）

### 1️⃣ 核心概念精确定义（变量）

1. 市场分层（板块结构）
    - 定义：按上市标准、投资者适当性、涨跌幅约束等形成的子市场集合（主板/创业/科创/北交等）。
    - 你真正用到的意义：决定“涨跌幅上限+交易/准入门槛+流动性结构”，从而改变可用策略与尾部风险形态。
2. 交易制度（微观结构）
    - 定义：交易时段、集合竞价规则、撮合优先级、涨跌幅、退市/ST、信息披露等对订单与价格形成的约束集合。
    - 意义：决定“哪些风险是可控（盘中）/不可控（开盘与隔夜）”。
3. 参与主体（资金结构）
    - 定义：对价格形成有系统性影响的资金群体（散户/游资、公募、外资、保险社保、券商自营与私募量化、上市公司行为、监管与中介）。
    - 意义：不同主体的周期与约束不同，导致不同波动来源与风格切换路径。
4. 流动性与路径风险
    - 定义：流动性=在可接受冲击成本下成交的能力；路径风险=在涨跌幅约束下，风险从“单日大跌”转移为“隔夜跳空/连续一字/流动性枯竭导致无法退出”。
5. 标的可交易寿命（ST/退市风险）
    - 定义：标的在制度与基本面约束下维持可连续交易/可退出的能力；ST/退市是制度化的风险标签与处置路径。

### 2️⃣ 机制拆解（两层以上因果链）

机制链 1：分层 → 约束集合 → 参与者筛选 → 价格路径

- 分层差异（涨跌幅/门槛/标的特征）
    
    → 筛选风险偏好与资金类型
    
    → 形成不同的波动分布（连板/急拉急跌/一字板/跳空）。
    

机制链 2：集合竞价规则 → 开盘定价 → 隔夜风险暴露

- 撤单限制/不成交阶段
    
    → 开盘瞬间成交不确定性上升
    
    → “开盘/隔夜风险”不能用盘中可撤单思维管理。
    

机制链 3：退市/ST → 尾部风险结构 → 技术信号失效

- 进入风险处置路径
    
    → 连续跌停+流动性枯竭
    
    → 技术入场信号无法保证退出；必须在“标的池规则”层提前过滤。
    

### 3️⃣ 理论前提与适用边界

- 前提：你交易的不是“价格”，而是“在制度约束下的价格路径”。
- 边界：
    - 框架能解释“风险从哪来、以什么形态出现”，不能替代“具体买卖规则与参数”。
    - 对单票的涨跌预测能力有限；对系统性风险（退市/流动性枯竭/隔夜跳空）识别能力更强。

### 4️⃣ 可迁移应用场景

- 任何交易系统的第一层：先写“场地规则+不可控风险清单”（你在哪些时间段、哪些品种、哪些板块上交易）。
- 建立统一的“标的池过滤”（ST/低流动性/高杠杆工具默认禁用）。

### 5️⃣ 最可能的认知偏误/薄弱点

- 把“涨跌幅上限”误当“风险上限”（路径风险更强）。
- 把“主体标签”当因果归因（看到直线拉升就断定量化/游资）。
- 把经验句子当规则（不落到条件+退出机制=不可复盘）。

### 6️⃣ 三句话结构压缩总结

1. 先搭市场地图：分层、制度、主体决定你能交易什么，以及风险以何种路径爆发。
2. 制度会改变风险暴露方式（尤其集合竞价与退市路径），技术信号无法覆盖这些尾部风险。
3. 任何“稳/稳赚”表述，只有落成“条件+失效位+退出机制”才允许进入系统。

## B｜方法论层：把 6–20 课压成“可执行系统骨架”（过滤器→触发器→风控器→复盘器）

### 1️⃣ 核心概念精确定义（系统零件）

- 题材（催化/叙事）：能在短期引发资金一致预期的方向。
- 股性（代理变量）：标的被短线资金反复选择的历史行为特征；材料用“涨停次数”做代理。
- 结构触发点：前高/平台/箱体上沿/颈线等“供需可能改变”的关键位。
- 趋势过滤器：用 MA20/MA60 或“高低点序列”过滤逆势。
- 量价确认：突破时成交活跃度上升且价格同步推进，用于降低假突破。
- 持仓线：用于把“拿不拿”变成机械动作的边界（MA5或结构位二选一）。
- 退出系统：固定止盈、移动止盈、结构止损、时间止损（至少两条同时存在）。
- 复盘标准：用次数、胜率、盈亏比、执行偏差把系统变成可迭代对象。

### 2️⃣ 机制拆解（两层以上）

机制链 A：题材 → 共识资金 → 龙头/核心形成 → 板块扩散

- 催化出现
    
    → 资金聚焦形成共识
    
    → 龙头率先突破/涨停
    
    → 梯队扩散产生赚钱效应
    
    → 情绪再强化。
    

机制链 B：过滤器→触发器→风控器的顺序约束

- 先过滤（题材/板块共振/股性/趋势）
    
    → 再触发（突破/晋级/首阴/回踩中的固定2个模型）
    
    → 再风控（失效位+止损口径）
    
    → 否则所有信号都会退化为“临盘解释”。
    

机制链 C：高潮→分化的退出逻辑

- 扩散到极值
    
    → 次日分化概率上升
    
    → 退出动作必须早于“确认退潮”，否则利润回吐吞噬盈亏比。
    

### 3️⃣ 执行步骤结构（最小可用版本）

注意：原材料参数大量缺失/不一致；以下只保留能落地的“结构口径”，其余列为待定。

步骤 0：场地与标的池（硬过滤）

- 目标函数：避免不可退出的尾部风险。
- 执行标准：ST/*ST 排除；低流动性标的排除；高杠杆工具默认禁用。

步骤 1：题材与板块过滤（只做主线）

- 目标函数：把机会集从“全市场”压缩到“当期资金共识”。
- 执行标准：当天主线题材明确；板块内存在共振梯队（非单票独舞）。

步骤 2：个股过滤（股性/核心梯队）

- 目标函数：提升“被资金再次选择”的概率。
- 执行标准：进入你固定口径的股性池（涨停次数阈值按板块分层设定）。
- 信息不足：阈值如何随行情调参缺约束，需你定义“调参频率与上限”。

步骤 3：入场触发（只保留 2 个模型，避免系统过杂）

- 目标函数：用可复盘触发器替代“看感觉”。
- 候选模型（从中固定2个）：
    
    1）突破触发：结构位突破 + 量价确认
    
    2）首阴触发：龙头首阴不破 MA5，次日转强确认
    
    3）二板晋级：首板后强度确认（需你补盘口/回封口径）
    
    4）回踩低吸：回踩 MA3/MA5 企稳（需你补“企稳确认”口径）
    
- 信息不足（必须标注）：
    - “突破位”统一定义缺失（前高/箱体/缺口/颈线需固定一种主口径）。
    - “爆量/量价齐升”缺阈值（相对均量/成交额口径待定）。
    - “转强/企稳”缺可观测标准（否则必然随意解释）。

步骤 4：持仓（用一条线把行为机械化）

- 目标函数：在趋势存在时避免过早下车；在失败时快速退出。
- 执行标准：持仓线二选一并固定：
    - MA5 持仓：收盘不破 MA5 继续；收盘破 MA5 进入退出
    - 结构持仓：不破突破位/平台支撑继续；破位收盘确认退出

步骤 5：止损（必须先于止盈确定）

- 目标函数：定义单笔的可承受最大错误。
- 执行标准：
    - 结构止损：跌回突破区/跌破关键结构位（收盘确认）→ 走
    - 或均线止损：收盘跌破 MA5 → 走
- 信息不足：止损用“盘中”还是“收盘”必须统一，否则复盘不可比。

步骤 6：止盈（固定比例 + 移动止盈二选一为主）

- 目标函数：锁利润同时保留趋势弹性。
- 可验证标准（来自材料但需你锁版本）：
    - 固定止盈：+15%~20% 减仓 50%；+30% 再减仓（比例需你固定）
    - 移动止盈：跌破 MA5（短）或 MA20（中长）收盘确认 → 卖出
- 信息不足：
    - “30%后回撤走”缺回撤阈值与触发口径。

步骤 7：复盘与训练（把系统变成可迭代对象）

- 目标函数：用统计替代故事。
- 执行标准（材料给的可用口径）：同一信号触发≥20次；胜率目标≥58%；盈亏比>1.8；记录执行偏差（追涨、扛单、浮盈加仓等）。

### 4️⃣ 隐含假设（必须显式化）

- 假设1：题材共识能形成可持续的资金流入，而不是一次性噪音。
- 假设2：突破+量价能显著降低假突破概率（需用你的数据验证）。
- 假设3：MA5/MA20能作为有效的“趋势边界”（不同波动标的可能需要不同敏感度）。
- 假设4：涨停次数能代表“股性/人气”（它是结果变量，可能在高波动末端误导）。

### 5️⃣ 适用范围与边界条件

- 更适用：主线清晰、情绪回暖、板块共振明显、量能能跟随的阶段。
- 高风险/不适用：退潮末期、系统性杀跌、极端缩量、政策突发巨量但次日无跟随的阶段。

### 7️⃣ 三句话结构压缩总结（方法论层）

1. 用四段结构做交易：先过滤题材与板块共振，再用固定触发器入场，最后用失效位与退出系统管理风险。
2. 持仓与退出必须机械化（MA5/结构位二选一），否则所有信号都会退化为临盘情绪。
3. 任何参数与胜率结论必须带回测口径；缺口（突破/量能/转强定义）不补齐就不能称为系统。
