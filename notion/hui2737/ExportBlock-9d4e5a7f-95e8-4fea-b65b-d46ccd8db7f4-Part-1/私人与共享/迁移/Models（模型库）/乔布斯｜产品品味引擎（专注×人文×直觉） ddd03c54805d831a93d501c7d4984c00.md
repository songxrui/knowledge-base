## ① 一句话模型

产品价值 = 极致取舍 × 人文品味 × 端到端体验 × 直觉验证。 [C]

## ② 专注：拒绝机

- 战略不是加法，是删掉 70–90% 看似不错的东西。 [C]
- 同一时期只推 3–4 个真正重要项目；其余进入拒绝清单。 [C]
- 判断标准：是否服务主线体验？不能服务则删。 [C]

## ③ 产品第一性

| 维度 | 乔布斯式问句 | 动作 |
| --- | --- | --- |
| 功能 | 它怎样工作？ | 减少认知摩擦 |
| 美学 | 它是否让人愿意靠近？ | 统一版式、节奏、质感 |
| 端到端 | 用户从看到到使用是否顺滑？ | 检查标题、正文、购买、交付 |
| 人文 | 它放大人的什么能力？ | 服务创造力而非炫技 |

## ④ RDF 的安全边界

- RDF 可用于把团队拉出“默认不可能”的惯性。 [C]
- RDF 不能替代事实、用户反馈与工程验证。 [C]
- 缺少品味和证据时，RDF 会退化成传销式自欺。 [C]

## ⑤ 向死而生

- 若连续多天回答“今天不是我想过的最后一天”，必须调整主线。 [C]
- 死亡问题用于删掉外部期待、虚荣排名和无意义忙碌。 [C]
- 连点成线只能向后看；当下只需高质量地打点。 [C]

## ⑥ if-then 卡

- if 项目过多 → 砍到 3 个以内；其余冷冻 30 天。 [C]
- if 内容只有信息没有体验 → 重写为“问题—路径—使用感”。 [C]
- if 想做市场调研 → 先做自己极度想用的最小版本，再给真实用户看。 [C]
- if 直觉与事实冲突 → 事实优先，直觉降级为假设。 [B]
- if 对外部期待恐惧 → 用“最后一天”问题重排优先级。 [C]
