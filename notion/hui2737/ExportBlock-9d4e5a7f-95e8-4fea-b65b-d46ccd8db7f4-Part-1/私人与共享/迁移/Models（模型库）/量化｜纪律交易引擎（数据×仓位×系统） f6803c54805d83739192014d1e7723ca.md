Anti-Use: 样本太小、数据不可得、规则无法执行时，不应伪量化；不要把相关性包装成因果。

Failure Modes: 只看胜率不看赔率；无仓位管理；回测过拟合；系统信号被情绪推翻；样本太小就放大；把量化当神谕。
Last Verified: 2026年5月10日
Leverage Type: Code
Model Type: Investment
Reuse Count: 0
Steps: 1) 定义Edge
2) 记录样本
3) 设Gate/Trigger/Sizing/Exit
4) 单笔风险≤可承受
5) 周复盘偏差并迭代
When to Use: 交易系统、内容运营、A/B 测试、风控、情绪剥离时。用数据寻找 edge，用仓位管理避免出局，用系统化规则代替盘感。

<aside>
📊

**量化｜纪律交易引擎**：别相信盘感，先定义 edge、仓位、退出和复盘。胜率不重要，长期不出局更重要。

</aside>

## ① 一句话模型

长期收益 = edge × 大数样本 × 仓位纪律 × 情绪剥离 × 复盘迭代。 [B]

## ② 数据驱动

- 不问自己想不想涨，问规则是否触发。 [B]
- 小优势必须靠大量样本放大；样本少时禁止自信。 [B]
- 相关性可用于试错，但不能伪装因果。 [B]

## ③ 交易流水线

| 模块 | 问题 | 动作 |
| --- | --- | --- |
| Gate | 什么条件下才允许看？ | 不满足直接跳过 |
| Trigger | 何时入场？ | 只执行规则信号 |
| Sizing | 亏多少还能活？ | 单笔风险≤账户可承受 |
| Exit | 何时退出？ | 触发即走，不解释 |

## ④ 凯利与半凯利

- 正期望机会也会因仓位过大而破产。 [A]
- 凯利公式给理论仓位；现实用半凯利或更保守，吸收估算误差。 [B]
- 仓位管理比单次预测更接近生存核心。 [A]

## ⑤ 迁移到内容/IP

- 每篇内容 = 一次实验，不是人格审判。 [C]
- 记录标题、封面、钩子、完读、收藏、转化。 [C]
- 放大有效因子，删除无效因子；不要靠情绪解释数据。 [C]

## ⑥ if-then 卡

- if 无 Gate → 不交易/不发布策略化实验。 [B]
- if 想临场改规则 → 记录冲动，盘后复盘。 [B]
- if 连续亏损 → 降低仓位或暂停，不加码。 [A]
- if 数据样本 <30 → 只当假设，不当结论。 [B]
- if 内容没人看 → 改因子，不攻击自己。 [C]
