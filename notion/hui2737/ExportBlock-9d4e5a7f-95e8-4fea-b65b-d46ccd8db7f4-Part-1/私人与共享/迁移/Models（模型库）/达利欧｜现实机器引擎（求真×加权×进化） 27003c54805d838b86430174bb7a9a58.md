# 达利欧｜现实机器引擎（求真×加权×进化）

Anti-Use: 需要心理安抚或隐私边界很高的场景，不宜过度透明；单人小事不必过度算法化。
Created: 2026年5月15日 12:15
Failure Modes: 把透明变成羞辱；把可信度等同名气；平均所有意见；诊断没做就开药；原则只写不执行；用系统化逃避现实行动。
Last Verified: 2026年5月10日
Leverage Type: Code
Model Type: Decision
Reuse Count: 0
Steps: 1) 记录事实而非感受
2) 找痛苦对应的机器 bug
3) 信息源可信度加权
4) 目标-问题-诊断-设计-执行
5) 把有效规则写成原则
When to Use: 复盘、团队决策、AI 协作、信息源筛选、个人系统修复时。用痛苦定位机器 bug，用可信度加权筛选观点，用五步流程改造系统。

## ① 一句话模型

结果 = 目标机器 × 反馈质量 × 可信度加权 × 迭代执行。 [B]

## ② 求真协议

- 极度求真：先写现实，不写愿望。 [B]
- 极度透明：让关键事实进入桌面；但不把透明变成羞辱。 [C]
- 痛苦是信号：不是自责材料，而是机器 bug 提示。 [B]

## ③ 可信度加权

| 信息源 | 权重 | 判断 |
| --- | --- | --- |
| 成功 ≥3 次且能解释机制 | 高 | 可进入决策 |
| 有一手实践但样本少 | 中 | 小规模试 |
| 只会讲观点/卖焦虑 | 低 | 不作为依据 |
| 无代价、无记录、无复盘 | 极低 | 过滤 |

## ④ 五步进化流程

1. 目标：我真正要什么？ [B]
2. 问题：什么阻碍目标？不许美化。 [B]
3. 诊断：根因是什么？先诊断再开药。 [B]
4. 设计：改哪条规则、环境或流程？ [B]
5. 执行：设时间、产出物、验收标准。 [B]

## ⑤ 机器视角

- 目标是产线终点；日程、工具、环境、奖励是机器零件。 [B]
- 行为失败优先改机器，不先骂人。 [B]
- 每次重大错误沉淀 1 条原则，否则错误会复发。 [B]

## ⑥ if-then 卡

- if 冲突发生 → 先写事实、解释、需求三列。 [C]
- if 多人意见冲突 → 按相关领域可信度加权，不平均。 [B]
- if 痛苦重复出现 → 找机器 bug，不写情绪散文。 [B]
- if 原则不能改变下一次动作 → 删除或改写。 [C]
- if 复盘没有 7 天实验 → 视为无效复盘。 [C]
