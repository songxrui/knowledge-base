# 【芒格宝典】专题一：多元思维模型与格栅理论

Anti-Use: 简单确定性问题（例如纯流程执行、已知标准答案）不必上格栅；避免过度分析导致拖延。
Created: 2026年5月15日 12:15
Failure Modes: 把格栅当阅读材料不产出；模型太多不收敛；只挑顺眼模型=确认偏误；不写反证与出局线；用术语堆砌代替行动。
Last Verified: 2026年5月9日
Model Type: Learning
Reuse Count: 0
Steps: 1) 定题型与目标函数（决策/学习/执行/下注）
2) 选 3–7 个学科透镜（概率/复利/激励/进化/机会成本/安全边际/反馈）
3) 让模型交叉验证：至少2个模型指向同一结论
4) 写出关键约束与出局线
5) 输出一张卡：结论 + 反证 + 下一步实验
When to Use: 用于处理复杂问题与跨域决策：避免“手里只有锤子”；用多学科少量关键模型交叉校验，输出可行动的结论与反证。

<aside>
🕸️

**芒格·格栅卡**：复杂问题不要用单一学科硬解；用少量关键模型交叉校验，让结论可行动、可反证。

</aside>

| 步骤 | 做什么 | 产出物 |
| --- | --- | --- |
| 1. 定题型 | 这是决策/学习/执行/下注哪一种？目标函数一句话是什么？ | 一句话目标函数 |
| 2. 选透镜 | 从常用关键透镜里选 3–7 个：复利、概率、激励偏差、机会成本、进化、临界质量、安全边际、反馈回路 | 透镜清单 |
| 3. 交叉验证 | 至少 2 个模型指向同一方向才算“信号”；否则降级为试错 | 一致结论/不一致点 |
| 4. 找关键约束 | 瓶颈变量是什么？出局线是什么？ | 关键约束 + 熔断 |
| 5. 强制输出卡 | 结论 + 反证（成立即不做） + 下一步最小实验 | 可复用决策卡 |

<aside>
✅

**判断句**：模型越少越好，但必须能交叉校验并产出行动。

</aside>
