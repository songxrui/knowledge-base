## ① 一句话模型

自由 = 清晰目标 × 删除低效 × 标准件 × 恐惧可修复 × MED。 [C]

## ② DEAL 流程

| 步骤 | 问题 | 产出物 |
| --- | --- | --- |
| D 定义 | 最低可接受生活成本是多少？ | TMI + 3 项支出 |
| E 删除 | 哪个时间黑洞无产出？ | 删除清单 + 释放小时 |
| A 自动化 | 哪个重复劳动可标准化？ | 模板/SOP/脚本 |
| L 解放 | 哪些地点/时段依赖可解除？ | 移动办公测试 |

## ③ 恐惧设定

- Define：最坏情况具体写，不写“完蛋”。 [C]
- Prevent：每个灾难对应 1 个预防动作。 [C]
- Repair：若发生，如何恢复到可接受状态？ [C]
- Cost of Inaction：不行动 6 月/1 年/3 年的代价。 [C]

## ④ MED 协议

1. 达成信号：指标 + 阈值 + 观察窗口。 [C]
2. 最小路径：关键动作 ≤2 个。 [C]
3. 频率剂量：写清每天/每周多少。 [C]
4. Stop Rule：达标即停，转投下一瓶颈。 [C]

## ⑤ 反忙碌红线

- 只有时长没有产出物 → 无效。 [C]
- 整理系统但不交付 → 伪进步。 [C]
- 自动化未稳定流程 → 放大混乱。 [C]
- 每周没有删 1 个时间黑洞 → 系统会膨胀。 [C]

## ⑥ if-then 卡

- if 任务太大不启动 → 写最坏情况/预防/修复。 [C]
- if 一周很忙但无产出 → 删除最大时间黑洞。 [C]
- if 目标模糊 → 先写达成信号，不写计划。 [C]
- if 已达标仍加码 → 停；转投瓶颈。 [C]
- if 想自动化 → 先做 3 次人工流程再封装。 [C]
