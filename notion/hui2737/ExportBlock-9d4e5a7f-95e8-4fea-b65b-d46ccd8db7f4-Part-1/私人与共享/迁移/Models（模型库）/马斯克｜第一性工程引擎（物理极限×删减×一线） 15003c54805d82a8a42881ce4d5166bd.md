## ① 一句话模型

创新 = 底层事实拆解 × 大胆删减 × 快速试错 × 一线校验；自动化必须最后上。 [B]

## ② 第一性原理三步

1. **列假设**：把“行业惯例、经验、别人都这么做”写出来。 [B]
2. **拆到底层**：物理、数学、材料、时间、用户注意力、真实成本。 [B]
3. **重新组装**：从底层事实反推更短路径，而非微调旧路径。 [B]

## ③ 五步算法

| 顺序 | 动作 | 禁忌 |
| --- | --- | --- |
| 1 | 质疑每个要求 | 不接受“部门说” |
| 2 | 删掉零件/流程 | 删不够就无法重构 |
| 3 | 简化和改良 | 先删后改 |
| 4 | 加快节拍 | 确认必要后再提速 |
| 5 | 自动化 | 不能自动化垃圾流程 |

## ④ 白痴指数

- 白痴指数 = 最终成本 / 基础原料或底层成本。 [B]
- 指数很高 → 中间层存在垄断、低效、包装或错误设计。 [B]
- 在内容/AI 中，若 80% 时间花在排版、找图、搬运，说明流程白痴指数过高。 [C]

## ⑤ 一线原则

- 不靠 PPT 判断系统；到代码、数据、用户、生产线现场看。 [B]
- 工厂才是产品：真正要建的是“持续产出好结果的机器”。 [B]
- 愿景可提供能量，但不能替代工程细节。 [B]

## ⑥ if-then 卡

- if 任务看似不可能 → 先问是否违背物理/数学底线；不违背则拆。 [B]
- if 流程复杂 → 先删 30%，再谈提速。 [B]
- if 想自动化 → 先证明流程本身必要且足够简单。 [B]
- if 项目只停在规划 → 去一线产出 1 个最小实物。 [C]
- if 想 All-in → 先过塔勒布不出局线；别复制高风险人格。 [B]
