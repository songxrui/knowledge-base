# 【格雷厄姆宝典】专题二：创作者作息与“做不规模化的事”

Created: 2026年5月15日 12:15
Failure Modes: 用管理者作息切碎Maker时间；把社交当推进；追求规模化曝光跳过对话；只优化系统不做作品
Last Verified: 2026年3月27日
Model Type: Learning
Reuse Count: 0
Steps: 1) 保护Maker时间块（连续≥90min）
2) 每天只推进一个可见交付（草稿/代码/模板）
3) 做不规模化的事：找3个真实用户对话
4) 用反馈改下一次交付
5) 避免会议/碎片化侵蚀
When to Use: 需要进入深度工作/创造者作息；内容或产品冷启动；想用“少数高质量对话”替代规模化曝光

【材料识别】方法论型为主（两套协议：半天深度块+不规模化触达），次为理论框架型（Maker/Manager粒度与切换成本）。

【核心概念】Maker Schedule=以半天为最小单元的深度排程；深度块=连续无打断窗口；切换成本=恢复到高密度所需分钟数；不规模化动作=早期手工高触达换真实反馈；痛点词典=10次对话原话集合。

【机制链】(1) 半天保护→切换减少→状态更深→产出质量上升；(2) 手工触达→真实痛点/付费语言→迭代收敛→冷启动转化提高。

【执行（目标函数/DoD）】Maker：半天只放1件深度事；DoD=每周≥3个无打断半天块。沟通集中到固定时段；DoD=记录破窗次数。会前2小时禁插会；DoD=临时会一律推迟到沟通窗口。不规模化：找10个高相关对象（写筛选理由）；DoD=名单+理由。3问访谈（最痛/替代/愿付费结果）；DoD=10条记录保留原话。按反馈迭代并复跑；DoD=1版痛点词典+1个迭代结论。

【隐含假设】你确有深度任务；可触达目标对象。

【边界】当天以协调/碎片执行为主时不必强行半天块。

【迁移】内容冷启动（对话生成选题措辞）；产品验证（对话→词典→迭代）；学习研究（半天块做深度输入+产出）。

【薄弱点】把90分钟当深度块（只够预热）；只发内容等算法不触达；样本选择偏差。

【3句总结】深度工作用半天单位保护，不与沟通混排。冷启动靠不规模化触达，让方向被真实语言校准。用DoD锁执行，否则必退化。
