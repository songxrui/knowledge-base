## 8 条死刑红线（命中 = REJECTED 重写）

1. 无具体数字/名词/我的真实案例/可验证来源 → REJECTED
2. 出现禁用词黑名单 → REJECTED
3. TOOL_LEDGER 目标skill有效调用<3次 → 该轮作废
4. 改写后字数 < 原文1.5倍且质量分<9.0 → 继续扩充
5. 条目未标注相关条目ID（孤点）→ REJECTED
6. 同一母题下条目<3条 → 从信源补充
7. AI三段式/值得注意的是/综上所述/在这个时代 → 重写
8. TOOL_LEDGER有记录但成品看不到skill输出痕迹 → 删除重跑

赋能 抓手 闭环 底层逻辑 本质上 综上所述 众所周知 不难发现

## 三阶段手术流程

### Phase 1 — 诊断（只读不写）
A. 通读 `D:\KnowledgeBase\cards\` 全条目，按母题归类
B. 初始质量分（0-10）：具体性+Skill痕迹+主题联系+无AI味+可发布性
C. 输出 `DIAGNOSIS_REPORT.md`：ID+分数+病灶+优先级
D. 汇报统计

### Phase 2 — 手术（逐条重构）
Step 1: 从 `weread_extracts\` / Notion导出 / AI会话找 ≥2 具体证据
Step 2: 调用 ≥1 内容创作skill，记入 `TOOL_LEDGER.md`
Step 3: 改写执行（去空洞+加核心论点+反共识+实操）
Step 4: 标注 ≥2 相关条目ID，填入 `CONNECTION_MATRIX.md`
Step 5: 质量复审<9.0 → 返回Step1

### Phase 3 — 重建整合
F. 母题综述卡（≥800字）+ 内部引用 + 横向联系
G. `CONNECTION_MATRIX.md` 零孤点 + mermaid连接图
H. `QUALITY_INDEX.md` 改写前后对比
I. `FINAL_AUDIT.md` 逐条核查8条红线
