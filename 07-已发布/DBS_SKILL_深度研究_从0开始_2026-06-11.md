## 二、逐项深研

### 2.1 dbs-content-system（内容结构化系统）

**compile-and-verify Phase1 编译**:
- 表层任务: 把本地大量内容变成结构化工程
- 深层目标: 让旧内容从"堆在文件夹里"变成"可复用/可追溯/可重组的资产"
- 第一性变量: 审计→结构→样本→规则稳定→批量→全量
- 成功标准: 系统"能用"（7个交付物齐全），不是"全量处理完"
- 主要失败模式: 跳过审计直接开工、结构未稳定就全量推进

**关键边界**:
- 处理: 大量文稿/推文/公众号/选题/案例/课程稿
- 不处理: 单篇润色/标题优化/短视频开头/少量零散素材

**7阶段递进**:
```
Phase1 工程初始化 → Phase2 素材审计 → Phase3 规则层建立
→ Phase4 内容单元抽取(样本模式) → Phase5 结构验证
→ Phase6 关系+去重+总览 → Phase7 批量→全量升级
```

**批量→全量升级闸门**（极严格）:
- 连续2批次无改字段规范
- 连续2批次无改关系规则
- 连续2批次无改去重规则
- 连续2批次无大面积返工
- 人工抽查30单元，误判≤3个

**董辉当前状态**: 已通过Phase1-6，235源文件→1543单元+305装配稿。待人工复核和关系索引补建。

### 2.2 dbs-content（内容创作诊断）

**核心哲学**（5原则）:
1. **文字洁癖是底线**: AI味问题不是AI的问题，是人不关心内容好不好
2. **精神控制是本质**: 封面+标题=认知劫持，特定文字排列触发特定神经机制
3. **内容好坏=投入精力×正确理解**: 新手越应该推高成本做内容
4. **先产品后内容**: 内容为产品服务，不是为自嗨
5. **知识博主核心工作只有两个**: 搞清楚+说清楚

**5维度诊断**:
- 选题诊断: 有没有具体的"敌人"？有没有切入点的锐度？
- 表达诊断: 封面+标题是否独立工作？文字是否口语友好？
- 结构诊断: 前5秒有没有Hook？节奏有没有起伏？
- 证据诊断: 有没有数字/故事/金句/权威背书/痛点共鸣？
- 行动诊断: 看完后用户知道下一步做什么吗？

**关键案例数据**:
- 改封面标题（内容不变）→ 播放量涨3倍，互动涨3倍，评论涨4倍
- 小红书起号48h 3200粉: 账号×内容×运营
- 24h售出第一个智能体: ¥9.9，先有产品后做内容

**对董辉的直接应用**:
- 用5维度诊断110篇文章 → 找最薄弱维度
- 所有文章开头检查: 5秒内有没有Hook?
- 先确定产品（¥99社群/电子书），再围绕产品做内容

### 2.3 dbs-diagnosis（商业模式诊断）

**核心公理**（6条）:
1. 商业模式是独立于人的客观存在 — 人是喂料员
2. 商业模式决定人的道德 — 好模式逼你做好人
3. 智力不直接变现，商业模式才变现 — 赚钱=执行力+商业模式
4. 流量≠收入 — 99%情况流量越大越不赚钱
5. 定价即产品 — 引流款和利润款价差最好10倍
6. 99%创业问题是心理问题 — 人们刻意选择"不知"来让自己"不行"

**两种模式**:
- **问诊**: 带着问题来→5层消解漏斗→问题被消解（不是回答）
- **体检**: 没有具体问题→3阶段拆解→完整诊断报告

**5层消解漏斗**:
1. 语言陷阱检测（25%问题在这里被消解）— "适合""值得""应该"等模糊词
2. 前提检测 — 问题成立的前提对吗？
3. 归因检测 — 因果方向对吗？
4. 边界检测 — 问题是商业问题还是心理问题？
5. 行动检测 — 真正阻挡行动的是什么？

**关键数据**: 8000+付费咨询，0.9%被真正解答，99.1%被消解

**对董辉的直接应用**:
- 当前最需诊断: "怎么从110篇文章开始赚钱？"→ 用消解漏斗拆解
- 检查是否有语言陷阱: "适合我的变现方式"中的"适合"需要定义
- 产品先行: 先确定一个可付费的产品，再判断商业模式

### 2.4 dbs-benchmark（对标分析）

**核心信条**（4条）:
1. 排除自我是决策加速器 — 所有关于"我"的讨论都是决策噪音
2. 0到1阶段，模仿是正确答案 — 差异化是后话，先活下来
3. 模仿的颗粒度决定模仿的质量 — 对标女主播裙子上3个线头你也得有3个
4. 高利润是唯一标准 — 高毛利/高复购/高壁垒/高增长都不等于高利润

**五重过滤法**:
1. 他赚钱吗？（利润至少是你预期的10倍）
2. 你能看懂吗？（获客→转化→交付→复购的完整链条）
3. 你能模仿吗？（有能力执行他的流程）
4. 排除自我（不讨论现有业务/资源/经历/偏好/兴趣）
5. 不讨论业务本质（能干就执行，不能干就换）

**对董辉的直接应用**:
- 对标dontbesilent: 5个筛子全部通过 ✅
- 对标姜胡子: 5个筛子全部通过 ✅
- 对标半佛仙人: 筛子2(看懂)可能困难 — 他的核心竞争力是"讨人喜欢的能力"
- 执行检查: 模仿颗粒度够吗？dontbesilent日更推文→董辉周更公众号→差距

### 2.5 dbs-hook（短视频开头优化）

**核心信条**（4条）:
1. 开头是内容的试用装，不是标题的延续
2. 好开头 = 话题 + Hook + 可信度
3. 开头要制造悬念，不要直接给答案
4. 开头必须口播友好

**工作流程**:
1. 接收文案
2. 诊断内容质量（关键！）— 5维度素材丰富度检查
3. 诊断当前开头 — 独立性/Hook/悬念/可信度/口播/匹配度
4. 生成优化方案

**Hook诊断维度**:
- 独立性: 不看标题能理解吗？
- Hook: 前5秒有抓手吗？
- 悬念: 有没有直接给答案？
- 可信度: 为什么听你讲？
- 口播友好: 能直接念出来吗？
- 匹配度: 开头和内容一致吗？

**对董辉的直接应用**:
- 所有公众号文章开头检查：前2句话有没有Hook？
- 改写建议: "今天聊XX" → 换成数据/反共识/故事开头
- 素材替代: "很多人认为XX" → 换成具体数字或案例

## 三、5个Skill的横向连接

### Skill间的调用关系
```
dbs-content-system (结构化)
  → 产出: 内容单元 + 装配稿
    → dbs-content (内容诊断)
      → 发现: 选题问题
        → dbs-good-question (问题说明)
      → 发现: 对标不清
        → dbs-benchmark (对标分析)
      → 发现: 变现迷茫
        → dbs-diagnosis (商业模式诊断)
      → 发现: 开头弱
        → dbs-hook (开头优化)
      → 发现: 行动障碍
        → dbs-action (执行力诊断)
```

### 共同底层哲学
所有DBS skill共享的核心假设:
1. **反空泛**: 每个结论必须绑定具体动作/数字/案例
2. **反讨好**: 内容不行就直接说不行
3. **先产品后内容**: 赚钱逻辑优先于创作逻辑
4. **模仿优先于创新**: 0到1阶段，抄到一模一样再谈差异化
5. **问题消解优于问题回答**: 大部分问题本身是错的
